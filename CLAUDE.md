# antel ŌĆö project guide

SRE Ansible automation repo. Ansible (project/playbook layout) + Python (custom modules,
filter plugins, helper scripts). Toolchain pinned via **uv** on Ubuntu 26.04 LTS.

**Dev env:** work inside the **devcontainer** (`.devcontainer/`). Host needs only a
container engine (Podman); `uv`, Ansible, Molecule, targets, and Jaeger live inside via
the `docker-in-docker` feature.

**Project goal:** explore the output of Ansible's OpenTelemetry callback
(`community.general.opentelemetry`, enabled in `ansible.cfg`). Playbooks run against
throwaway **docker** targets (Molecule, via docker-in-docker); emitted traces are viewed
in **Jaeger** (`observability/compose.yaml`). Canonical Workshop/LXD was evaluated and
dropped ŌĆö over-engineered for this goal (no LXD-target interface; nesting friction).

## Structure

- `inventories/<env>/` ŌĆö one dir per environment (`production`, `staging`), each with its own
  `hosts.yml`, `group_vars/`, `host_vars/`. **No global inventory in `ansible.cfg`** ŌĆö always
  pass `-i inventories/<env>/hosts.yml` so environments never mix.
- `playbooks/` ŌĆö top-level plays (`site.yml` is the entrypoint).
- `roles/` ŌĆö one role per service/concern (`tasks/`, `handlers/`, `defaults/`, `vars/`,
  `templates/`, `meta/`).
- `library/`, `filter_plugins/` ŌĆö custom Python for Ansible (auto-discovered via `ansible.cfg`).
- `molecule/default/` ŌĆö Molecule scenario; docker driver, throwaway target containers.
- `observability/` ŌĆö `compose.yaml` for the Jaeger all-in-one OTLP viewer.
- `tests/` ŌĆö pytest for Python helpers.

The OTel callback runs on the **control node** (where ansible executes), exporting to
`OTEL_EXPORTER_OTLP_ENDPOINT` (default `http://localhost:4317`, the Jaeger container).

## Commands

Everything runs through `uv run` (see `README.md` for the full list). Key ones:

```bash
uv sync                                                   # install pinned toolchain
uv run yamllint . && uv run ansible-lint                  # lint YAML + Ansible
uv run ruff check . && uv run ruff format && uv run mypy . # lint/format/type-check Python
uv run ansible-playbook -i inventories/<env>/hosts.yml playbooks/site.yml --check --diff
uv run molecule test                                      # docker targets: createŌåÆconvergeŌåÆverifyŌåÆdestroy
uv run molecule converge                                  # apply playbook, keep containers, emit OTel
uv run pytest                                             # Python helper tests
docker compose -f observability/compose.yaml up -d        # Jaeger viewer (UI :16686, OTLP :4317)
```

## Standards & conventions

### Ansible / YAML
- **Idempotency first** ŌĆö every task safe to re-run. Prefer modules over `command`/`shell`;
  if shell is unavoidable, guard with `changed_when` / `creates` / `when`.
- **FQCN always** ŌĆö `ansible.builtin.copy`, not `copy`.
- **Name every task** with a clear, action-describing string.
- **No plaintext secrets** ŌĆö use Ansible Vault or an external secrets manager; never commit
  unvaulted credentials (`.gitignore` covers `*.vault`, `vault_pass*`, `.env*`).
- Standard role layout; keep `defaults/` overridable, `vars/` for constants.
- Use **handlers** + `notify` for restart-on-change, not unconditional restarts.
- Booleans as `true`/`false` (lowercase); quote version-like strings. Tag tasks for selective runs.
- Linting enforces the **production** profile (`.ansible-lint`); `.yamllint` for YAML hygiene.
- Sources: [Ansible playbook best practices](https://docs.ansible.com/ansible/latest/tips_tricks/ansible_tips_tricks.html),
  [ansible-lint](https://ansible.readthedocs.io/projects/lint/).

### Python
- **Ruff** for format + lint (`ruff format`, `ruff check --fix`); **mypy** strict for types.
  Type-annotate public functions (PEP 8 / 257 / 484).
- **uv** for env + deps (`uv sync`, `uv run`, `uv add`); deps pinned in `uv.lock`.
- **pytest** ŌĆö plain `assert`, fixtures, `@pytest.mark.parametrize`.
- f-strings; `pathlib` over `os.path`; `logging` over `print`; no mutable default args.
  Prefer stdlib + existing deps over adding new ones.
- Sources: [ruff](https://docs.astral.sh/ruff/), [uv](https://docs.astral.sh/uv/),
  [pytest](https://docs.pytest.org/).
