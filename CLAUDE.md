## Structure

- `inventories/<env>/` â€” one dir per environment (`production`, `staging`), each with its own
  `hosts.yml`, `group_vars/`, `host_vars/`. **No global inventory in `ansible.cfg`** â€” always
  pass `-i inventories/<env>/hosts.yml` so environments never mix.
- `playbooks/` â€” top-level plays (`site.yml` is the entrypoint).
- `roles/` â€” one role per service/concern (`tasks/`, `handlers/`, `defaults/`, `vars/`,
  `templates/`, `meta/`).
- `library/`, `filter_plugins/` â€” custom Python for Ansible (auto-discovered via `ansible.cfg`).
- `tests/` â€” pytest for Python helpers.

## Commands

Everything runs through `uv run` (see `README.md` for the full list). Key ones:

```bash
uv sync                                                   # install pinned toolchain
uv run yamllint . && uv run ansible-lint                  # lint YAML + Ansible
uv run ruff check . && uv run ruff format && uv run mypy . # lint/format/type-check Python
uv run ansible-playbook -i inventories/<env>/hosts.yml playbooks/site.yml --check --diff
uv run molecule test                                      # test roles on ephemeral hosts
uv run pytest                                             # Python helper tests
```

## Standards & conventions

### Ansible / YAML
- **Idempotency first** â€” every task safe to re-run. Prefer modules over `command`/`shell`;
  if shell is unavoidable, guard with `changed_when` / `creates` / `when`.
- **FQCN always** â€” `ansible.builtin.copy`, not `copy`.
- **Name every task** with a clear, action-describing string.
- **No plaintext secrets** â€” use Ansible Vault or an external secrets manager; never commit
  unvaulted credentials (`.gitignore` covers `*.vault`, `vault_pass*`, `.env*`).
- Standard role layout; keep `defaults/` overridable, `vars/` for constants.
- Use **handlers** + `notify` for restart-on-change, not unconditional restarts.
- Booleans as `true`/`false` (lowercase); quote version-like strings. Tag tasks for selective runs.
- Linting enforces the **production** profile (`.ansible-lint`); `.yamllint` for YAML hygiene.
- Sources: [Ansible playbook best practices](https://docs.ansible.com/ansible/latest/tips_tricks/ansible_tips_tricks.html),
  [ansible-lint](https://ansible.readthedocs.io/projects/lint/).

### Python
- **Ruff** for format + lint (`ruff format`, `ruff check --fix`); **mypy** strict for types.
  Type-annotate public functions (PEP 8 / 257 / 484).
- **uv** for env + deps (`uv sync`, `uv run`, `uv add`); deps pinned in `uv.lock`.
- **pytest** â€” plain `assert`, fixtures, `@pytest.mark.parametrize`.
- f-strings; `pathlib` over `os.path`; `logging` over `print`; no mutable default args.
  Prefer stdlib + existing deps over adding new ones.
- Sources: [ruff](https://docs.astral.sh/ruff/), [uv](https://docs.astral.sh/uv/),
  [pytest](https://docs.pytest.org/).
